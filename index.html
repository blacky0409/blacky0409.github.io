SYSTOR 2025 — Submission

A Picture Is Worth 1000 Approximations:
Emulating and Directly Analyzing Physical Fragmentation on Aged SSDs

SSLAB  ·  NVMeVirt  ·  SSD Aging  ·  Physical Fragmentation  ·  FTL
Overview - I should modify

SSD aging은 현실적인 스토리지 성능 평가에 필수적이나, aged device 상태를 재현하려면 며칠씩 연속 workload를 실행해야 합니다. 본 논문에서는 aged SSD의 완전한 내부 상태를 캡처하고 새로운 에뮬레이터 인스턴스에 즉시 복원하는 snapshot 기반 방법론을 제안합니다.

또한 상용 SSD의 블랙박스 특성으로 인해 접근 불가했던 physical fragmentation을 직접 분석할 수 있는 3-파라미터 SPF 모델을 설계했습니다.

Key Contributions
01 / Snapshot
SSD 상태 캡처 & 즉시 복원
13일이 걸리던 aging 과정을 Valid-Only dump로 수십 초 내 재현.
02 / SPF Model
Physical Fragmentation 정량 제어
Pdegree, Lburst, Nstreams 세 파라미터로 0~100% 재현 가능한 fragmentation 모델.
03 / Analysis
논리 & 물리 파편화 복합 분석
2×2 실험 매트릭스로 독립적·복합적 성능 영향 최초 정량화.
Key Results
21.2% seq. throughput drop 40.1% p99 latency increase QD=32 at peak vulnerability 13 days → seconds

KCC 2024

Tiered Memory 구조를 위한 LSM Tree 개선 연구

한국정보과학회 KCC 2024  ·  LSM Tree  ·  Tiered Memory  ·  Hot/Cold Data Separation
Overview

본 프로젝트는 CXL(Compute Express Link) 기술 발전으로 도입된 Tiered Memory 환경에서, 기존 LSM Tree 구조가 가지는 성능적 한계를 분석하고 이를 최적화한 새로운 아키텍처를 설계 및 구현한 연구입니다.

[그림 1] CXL를 통한 Tiered Memory 구조

최근 빠른 주 메모리(Fast Memory)와 용량이 큰 서브 메모리(Slow Memory)를 함께 사용하는 환경이 대두되었습니다. 하지만 기존의 Monolithic 구조 LSM Tree는 데이터 검색 시 속도가 느린 서브 메모리까지 탐색해야 하므로 이러한 계층형 환경에서 I/O 효율이 급격히 저하되는 한계가 있었습니다. 저는 이러한 하드웨어 구조적 한계를 소프트웨어 아키텍처 개선으로 해결하여 안정적인 성능을 확보하고자 했습니다.

Details

1. Key-Value 분리 및 아키텍처 슬림화

문제를 해결하기 위해 WiscKey 논문에서 제시된 'Key sorting과 가비지 컬렉션의 Decoupling' 아이디어를 시스템 코드로 직접 구현했습니다.

[그림 2] WiscKey Data Layout on SSD

입력 데이터에서 Key와 해당 Value의 물리적 주소(Location/Offset 및 Size)만을 LSM Tree에 저장하고, 실제 데이터 용량의 대부분을 차지하는 Value는 별도의 로그 파일(vLog)에 순차적으로 Append하는 방식을 취했습니다.

  • 핵심 이점: Value 데이터가 트리에서 제외됨으로써 LSM Tree의 전체 크기가 획기적으로 축소됩니다.
  • 성능 최적화: 작아진 트리 전체를 접근 속도가 매우 빠른 주 메모리 상에 온전히 Cache할 수 있어, 검색 시 발생하는 서브 메모리 탐색 오버헤드를 완전히 제거했습니다.
  • I/O 증폭 해결: 압축 과정에서 무거운 Value 데이터를 반복적으로 이동시킬 필요가 없어 Write Amplification 문제를 근본적으로 최소화했습니다.

2. Tiered Memory 인지형 동적 Garbage Collection

두 번째 핵심 기술은 데이터의 사용 빈도에 따라 메모리 배치를 최적화하는 '동적 로그 파일 Garbage Collection' 파이프라인 구축입니다.

[그림 3] Log file Garbage Collection

로그 파일은 tail과 head 포인터를 이용해 신뢰할 수 있는 구간만을 엄격히 추적합니다. GC 가동 시 Valid가 입증된 데이터만 head로 마이그레이션하여 보존하고, 참조가 끊긴 Invalid 데이터의 점유 공간은 포인터 전진을 통해 즉시 반환함으로써 스토리지 파편화를 막고 공간 효율을 극대화했습니다.

특히, LSM Tree 상위 레벨로 밀려난 데이터는 상대적으로 사용 빈도가 낮다는 점에 착안했습니다. 따라서 GC 과정에서 Cold 데이터는 용량이 큰 서브 메모리로 마이그레이션하고, 자주 사용하는 Hot 데이터는 주 메모리에 남겨둠으로써 한정된 주 메모리 자원을 가장 효율적으로 사용하는 데이터 순환 구조를 완성했습니다.

3. 부하 테스트 및 실험 결과

NUMA 시스템 환경을 구축하여 30개의 스레드로 100,000번의 입출력을 발생시키는 고부하 테스트를 통해 성능을 검증했습니다.

[그림 4] 기존 LSM Tree와 개선된 LSM Tree 성능 비교(데이터가 사라져 옛날 사진 첨부하였습니다..)

실험 결과, 모든 데이터가 주 메모리에 있는 'Best Case'와 서브 메모리에 있는 'Worst Case' 사이에서 약 20%의 성능 편차를 보이던 기존 구조와 달리, 새롭게 구현한 아키텍처는 메모리 계층에 관계없이 편차 없는 안정적이고 균일한 처리 속도를 입증했습니다.

NVMeVirt 다중 Instance 기능 및 성능 모델 개선 연구

한국정보과학회 KSC 2023  ·  NVMeVirt  ·  SSD Emulator  ·  Multi-Instance  ·  장려상 수상
Overview

차세대 스토리지인 ZNS SSD를 연구하는 데 있어 학계와 산업계는 보편적으로 NVMeVirt와 같은 커널 기반 에뮬레이터를 활용합니다. 하지만, 당시 NVMeVirt는 실제 하드웨어의 특성을 제대로 담아내지 못하는 치명적인 논리 오류와, 단일 디바이스에 종속된 아키텍처라는 두 가지 큰 문제를 안고 있었습니다. 구체적으로는 ZNS의 핵심 연산인 append와 write의 처리 시간을 동일하게 취급하는 오류가 있었고, 플래시 메모리의 물리적 특성인 블록 Erase 비용이 reset 연산에서 누락되어 시뮬레이션의 신뢰도가 현저히 떨어졌습니다. 또한 전역 변수에 의존하는 설계 탓에 대규모 시스템 환경을 모사하기 위한 다중 인스턴스 생성이 불가능했습니다.

Details

저는 이러한 맹점을 해결하기 위해 가장 먼저 물리적 하드웨어 특성을 반영하여 ZNS 성능 모델을 재설계했습니다. 기존 시스템에서 하나로 묶여 있던 append와 write의 처리 파이프라인을 분석해 분리하고, 실제 디바이스에서 append가 write보다 지연 시간이 길다는 점을 반영하여 명시적으로 latency를 증가시켰습니다. 이와 더불어 Samsung 980 PRO의 실제 Erase Latency 스펙을 참고해 기존에 누락되어 있던 reset 연산 비용을 새롭게 구현했으며, 단순 지연이 아닌 연산의 실제 실행 시간까지 실측하여 합산하는 정교한 모델링을 완성했습니다.

나아가 대규모 병렬 테스트가 불가능했던 단일 인스턴스의 제약을 허물기 위해 C언어 기반의 커널 모듈을 대대적으로 리팩토링했습니다. 단일 디바이스에 맞춰 정적(Static)으로 할당되던 리소스들을 모듈화하고, Global Variable에 의존하던 방식을 로컬 변수 기반으로 전환하여 인스턴스 간 메모리를 완벽히 격리했습니다. 컴파일 타임에 결정되던 설정 역시 런타임 동적 로딩 구조로 유연하게 변경했습니다.

동시에 기존 procfs로 동작하던 사용자 인터페이스를 sysfs로 개편하여, 사용자가 /sys/nvmevirt/config 디렉토리 하위에서 다수의 가상 디바이스를 독립적으로 생성하고 제어할 수 있는 체계적인 환경을 구축했습니다.

이러한 코드 레벨의 치열한 아키텍처 개선 결과, 기존 시스템 대비 최대 256개의 독립적인 가상 인스턴스를 동시 구동할 수 있는 압도적인 시스템 확장성을 확보했습니다.

fio를 이용한 벤치마크 테스트를 통해 수많은 디바이스가 동시에 I/O를 발생시키는 극한의 부하 상황에서도 성능 하락을 방어하며 안정적으로 동작함을 증명했습니다.

또한 성능 모델 재설계를 통해 IOPS와 Bandwidth를 실제 상용 ZNS 디바이스 수준으로 끌어올리며 시뮬레이터의 신뢰도를 대폭 향상시켰습니다. 단일 논문 작성에 국한된 성과를 넘어, 시스템 엔지니어로서 오픈소스의 근본적인 구조를 개선한 이 프로젝트의 성과를 인정받아 한국정보과학회 KSC 2023에서 장려상을 수상하는 쾌거를 이룰 수 있었습니다.
